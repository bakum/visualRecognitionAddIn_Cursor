# VisualRecognition — нативная внешняя компонента 1С

Windows DLL для платформы **1С:Предприятие**: извлечение **текстового слоя** из PDF; опционально — эвристический разбор украинских первичных документов (рахунок на оплату, рахунок-фактура и т.п.) в **JSON (UTF-8)** либо возврат **плоского текста** без разбора структуры.

Лицензия исходного кода компоненты: **GNU AGPL v3** (см. заголовки файлов).

## Возможности

- **PDF → текст** через MuPDF (если доступен пакет `unofficial-libmupdf`) или **PDFium** (подтягивается CMake-скриптом; опционально встраивается в DLL с отложенной загрузкой).
- **Разбор текста** в структурированный JSON: контрагент, договор, банк, реквизиты, табличная часть позиций, полный исходный текст в `rawText`.
- **Только текст из PDF** (UTF-8), без парсинга первички — методы `ИзвлечьТекстИзPdf` / `ИзвлечьТекстИзPdfBase64` (те же коды `ПоследняяОшибка`, что у разбора).
- **Разбор через Gemini, PDF** (`РазобратьПервичныйДокументPdfИИ*`): вход — **PDF** (BLOB или Base64); страницы могут быть сканами и/или с текстовым слоем. Целевая структура JSON — как у локального разбора первички.
- **Разбор через Gemini, изображение** (`РазобратьПервичныйДокументИзображениеИИ*`): вход — **одно** растровое изображение (BLOB или Base64). Формат определяется по сигнатуре: **JPEG, PNG, GIF, WebP, BMP, TIFF**. **PDF сюда передавать нельзя** — для PDF используйте `РазобратьПервичныйДокументPdfИИ*`. Тот же JSON по схеме первички.
- **Ограничение (локальные методы PDF, не ИИ):** извлечение текста и эвристический разбор опираются на **текстовый слой** PDF. Отсканированные PDF без OCR **не** распознаются локально (код ошибки `2`).
- Имя класса компоненты (см. `GetClassNames`): **`VisualRecognition`**.

## Сборка (кратко)

- **CMake** ≥ 3.20, **MSVC**, **C++17**.
- Задайте **`BOOST_ROOT`** на каталог Boost (в `CMakeLists.txt` по умолчанию путь-пример; переопределите `-DBOOST_ROOT=...`).
- Соберите под нужную разрядность (x86 / x64). Имена выходных файлов: `visualAddInWin32.dll`, `visualAddInWin64.dll`.
- Опции CMake (Windows): `VISUAL_ADDIN_WITH_PDF`, `VISUAL_ADDIN_EMBED_PDFIUM` — см. `CMakeLists.txt`.

Артефакты для публикации описаны в **`MANIFEST.xml`** (bundle для подключения в 1С).

## Свойства

| Англ. (lang=0)   | Рус. (lang=1)           | Тип     | Описание |
|------------------|-------------------------|---------|----------|
| `Version`        | `Версия`                | Строка  | Версия компоненты (из CMake `PROJECT_VERSION`). |
| `LastErrorCode`  | `ПоследняяОшибка`       | Число   | Код результата последней операции: **`0` — успех**. |
| `LastErrorText`  | `ТекстПоследнейОшибки`  | Строка  | Текст ошибки (UTF-8); при коде `0` — пустая строка. |
| `AIStudioApiKey` | `КлючAPIAIStudio`       | Строка  | Ключ API [Google AI Studio](https://aistudio.google.com/) (UTF-8); **чтение и запись**. |
| `GeminiModel`    | `МодельGemini`          | Строка  | Идентификатор модели Gemini для всех ИИ-методов (`*ИИ*`, UTF-8), по умолчанию **`gemini-2.0-flash`**. Пустая строка после обрезки пробелов — снова подставляется значение по умолчанию. Допустимы латинские буквы, цифры, `.`, `-`, `_`. |

Остальные свойства **только для чтения**. Для ИИ: **`КлючAPIAIStudio`** и **`МодельGemini`** задаются из 1С (чтение и запись).

### Коды `ПоследняяОшибка` (`LastErrorCode`)

| Код | Значение |
|-----|----------|
| `0` | Успех (ошибок компоненты по результату вызова нет). |
| `1` | Пустые данные PDF **или** пустые данные изображения (ИИ). |
| `2` | Текст из PDF не извлечён (шифрование, скан без слоя, повреждённый файл). |
| `3` | Поддержка PDF отключена при сборке. |
| `4` | Некорректная строка Base64. |
| `5` | Неверный тип аргумента: ожидались двоичные данные PDF. |
| `6` | Неверный тип аргумента: ожидалась строка Base64. |
| `7` | Не задан ключ API Gemini (`КлючAPIAIStudio` / `AIStudioApiKey`). |
| `8` | Ошибка обращения к Gemini (сеть, HTTP, ответ API, разбор JSON); текст — в `ТекстПоследнейОшибки`. |
| `99` | Прочая ошибка; детали — в `ТекстПоследнейОшибки` / поле `error` JSON. |

## Методы

| Рус. (имя метода) | Описание |
|-------------------|----------|
| `РазобратьПервичныйДокументPdf` | PDF как **двоичные данные** (BLOB). |
| `РазобратьПервичныйДокументPdfBase64` | PDF в виде строки **Base64** (UTF-8). |
| `ИзвлечьТекстИзPdf` | Только **плоский текст** PDF (UTF-8); аргумент — **двоичные данные** (BLOB). |
| `ИзвлечьТекстИзPdfBase64` | То же; PDF в строке **Base64** (UTF-8). |
| `РазобратьПервичныйДокументPdfИИ` | Структурирование через **Google Gemini** (PDF как BLOB): в том числе **сканы** и PDF **с** текстовым слоем. Перед вызовом задайте **`КлючAPIAIStudio`**. |
| `РазобратьПервичныйДокументPdfИИBase64` | То же; PDF в строке **Base64** (UTF-8). |
| `РазобратьПервичныйДокументИзображениеИИ` | Структурирование через Gemini: **одно изображение** как BLOB (JPEG/PNG/GIF/WebP/BMP/TIFF по сигнатуре). |
| `РазобратьПервичныйДокументИзображениеИИBase64` | То же; изображение в строке **Base64** (UTF-8). |

**Возвращаемое значение**

- Методы **`РазобратьПервичныйДокументPdf*`**, **`РазобратьПервичныйДокументPdfИИ*`** и **`РазобратьПервичныйДокументИзображениеИИ*`** — строка UTF-8: JSON с полями результата или объект `{"error":"..."}` при ошибке конвейера.
- Методы **`ИзвлечьТекстИзPdf*`** — строка UTF-8 с извлечённым текстом; при ошибке — **пустая строка**, детали в **`ПоследняяОшибка`** / **`ТекстПоследнейОшибки`** (JSON с `error` для этих методов не возвращается).

При кодах **`5`** и **`6`** компонента дополнительно генерирует **исключение** (как при неверном типе параметра в 1С).

## Формат успешного JSON

Корневой объект (ключи на английском):

- `counterparty` — `supplier`, `buyer`, `inn`, `kpp` (и др. по мере заполнения парсером).
- `contract` — `numberAndDetails`, `date`.
- `invoiceNumber` — номер счёта / первички.
- `bankDetails` — `bankName`, `bik`, `correspondentAccount`, `settlementAccount`.
- `lineItems` — массив позиций; типичные поля элемента: `name`, `quantity`, `unit`, `price`, `amount`.
- `rawText` — полный текст, на основе которого выполнялся разбор.

Точный набор полей зависит от распознанного шаблона документа.

## Примеры для 1С (BSL)

Подключение из файлов на диске (пути и способ размещения DLL — по правилам вашей ИБ):

```bsl
ИмяКомпоненты = "VisualRecognition";
ПутьКDll = "C:\Путь\К\visualAddInWin64.dll"; // или Win32

Если Не ПодключитьВнешнююКомпоненту(ПутьКDll, ИмяКомпоненты, ТипВнешнейКомпоненты.Native) Тогда
    ВызватьИсключение "Не удалось подключить внешнюю компоненту";
КонецЕсли;

// Имя объекта зависит от версии платформы; часто используется схема AddIn.<Идентификатор>.<Класс>:
Компонента = Новый("AddIn.VisualRecognition.VisualRecognition");
// Если вариант не подходит — уточните в синтакс-помощнике по строке «Внешняя компонента» для вашей версии 1С.
```

Разбор PDF из двоичных данных (например, из `ДвоичныеДанные` файла):

```bsl
JsonСтрока = Компонента.РазобратьПервичныйДокументPdf(ДвоичныеДанныеФайла);

Если Компонента.ПоследняяОшибка = 0 Тогда
    // Разбор JSON на стороне 1С (XDTO, чтение вручную и т.д.)
    Чтение = Новый ЧтениеJSON;
    Чтение.УстановитьСтроку(JsonСтрока);
    Результат = ПрочитатьJSON(Чтение); // или своя логика
Иначе
    Сообщить("Ошибка " + Компонента.ПоследняяОшибка + ": " + Компонента.ТекстПоследнейОшибки);
КонецЕсли;
```

Разбор PDF из Base64:

```bsl
JsonСтрока = Компонента.РазобратьПервичныйДокументPdfBase64(Base64Строка);

Если Компонента.ПоследняяОшибка <> 0 Тогда
    ВызватьИсключение Компонента.ТекстПоследнейОшибки;
КонецЕсли;
```

Разбор через **Gemini** (ключ из [Google AI Studio](https://aistudio.google.com/); нужен исходящий **HTTPS**). Модель задаётся свойством **`МодельGemini`** (по умолчанию `gemini-2.0-flash`):

```bsl
Компонента.КлючAPIAIStudio = КлючСохраненныйВБезопасномМесте;
Компонента.МодельGemini = "gemini-2.0-flash"; // при необходимости другая, например gemini-2.5-flash
JsonСтрока = Компонента.РазобратьПервичныйДокументPdfИИ(ДвоичныеДанныеФайла);
// или: JsonСтрока = Компонента.РазобратьПервичныйДокументPdfИИBase64(Base64Строка);

// Отдельный файл-скан (JPEG/PNG и т.д.), не PDF:
JsonСтрока = Компонента.РазобратьПервичныйДокументИзображениеИИ(ДвоичныеДанныеКартинки);
// или: JsonСтрока = Компонента.РазобратьПервичныйДокументИзображениеИИBase64(Base64Картинки);
```

Извлечение текста без разбора первички (произвольный PDF с текстовым слоем):

```bsl
ТекстДокумента = Компонента.ИзвлечьТекстИзPdf(ДвоичныеДанныеФайла);
// или: ТекстДокумента = Компонента.ИзвлечьТекстИзPdfBase64(Base64Строка);

Если Компонента.ПоследняяОшибка = 0 Тогда
    // ТекстДокумента — UTF-8, дальнейшая обработка на стороне 1С
Иначе
    Сообщить(Компонента.ТекстПоследнейОшибки);
КонецЕсли;
```

Проверка версии:

```bsl
ВерсияКомпоненты = Компонента.Версия; // строка, напр. "1.0.0"
```

## Обработка ошибок

1. После вызова метода проверяйте **`ПоследняяОшибка` = 0** (или **`LastErrorCode`** при английских именах).
2. При ненулевом коде показывайте пользователю **`ТекстПоследнейОшибки`**.
3. Для методов **`РазобратьПервичныйДокументPdf*`** строка результата может содержать JSON вида `{"error":"..."}` — это согласовано с кодом в свойствах; при необходимости разбирайте и её. Для **`ИзвлечьТекстИзPdf*`** ориентируйтесь на **`ПоследняяОшибка`**, а не на JSON в возврате.

---

При необходимости OCR для сканов PDF потребуется отдельный этап (вне компоненты или доработка с рендером страниц и движком OCR).
