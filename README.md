# VisualRecognition — нативная внешняя компонента 1С

Windows DLL для платформы **1С:Предприятие**: структурирование первичных документов (счёт, счёт-фактура и т.п.) в **JSON (UTF-8)** через **Google Gemini** (включая **сканы** и PDF с текстовым слоем). Локального извлечения текста из PDF (MuPDF/PDFium) и методов «только текст» в сборке **нет** — нужен ключ **AI Studio** и исходящий **HTTPS**.

Лицензия исходного кода компоненты: **GNU AGPL v3** (см. заголовки файлов).

## Возможности

- **Разбор PDF** (`РазобратьПервичныйДокументPdf*`, `РазобратьПервичныйДокументPdfИИ*`): одно и то же — запрос к **Gemini**; BLOB или Base64.
- **Разбор изображения** (`РазобратьПервичныйДокументИзображениеИИ*`): одно растровое изображение (JPEG, PNG, GIF, WebP, BMP, TIFF по сигнатуре). **PDF сюда не передавать** — только методы с `Pdf`.
- Имя класса: **`VisualRecognition`**.

## Сборка (кратко)

- **CMake** ≥ 3.20, **MSVC**, **C++17**.
- Задайте **`BOOST_ROOT`** на каталог Boost (в `CMakeLists.txt` по умолчанию путь-пример; переопределите `-DBOOST_ROOT=...`).
- Соберите под нужную разрядность (x86 / x64). Имена выходных файлов: `visualAddInWin32.dll`, `visualAddInWin64.dll`.
- Зависимости: **Boost.JSON**, **WinHTTP**; отдельный **pdfium.dll** / **MuPDF** не требуются.

Артефакты для публикации описаны в **`MANIFEST.xml`**.

| `GeminiModel`    | `МодельGemini`          | Строка  | Идентификатор модели Gemini (UTF-8), по умолчанию **`gemini-3-flash-preview`**. Пустая строка после обрезки пробелов — снова подставляется значение по умолчанию. |
| `GeminiReceiveTimeoutMs` | `ТаймаутПолученияGeminiМс` | Число | Таймаут ожидания ответа Gemini на сетевом уровне (мс). По умолчанию `45000`. Применяется к каждому HTTP-вызову. |
| `GeminiTotalDeadlineMs` | `ОбщийДедлайнGeminiМс` | Число | Общий дедлайн одного вызова Gemini с учётом внутренних ретраев (мс). По умолчанию `65000`. |
| `LastPromptTokens` | `ПоследниеВходящиеТокены` | Число | Количество входящих токенов (`promptTokenCount`) из `usageMetadata` последнего вызова Gemini. |
| `LastOutputTokens` | `ПоследниеИсходящиеТокены` | Число | Количество выходных токенов (`candidatesTokenCount`) из `usageMetadata` последнего вызова Gemini. |
| `LastTotalTokens` | `ПоследниеВсегоТокенов` | Число | Общее количество токенов (`totalTokenCount`) из `usageMetadata` последнего вызова Gemini. |
| `LastGeminiUsageJson` | `ПоследняяСтатистикаТокеновJSON` | Строка | JSON-объект со статистикой токенов последнего вызова Gemini (`promptTokenCount`, `candidatesTokenCount`, `totalTokenCount`) или `{}` если данные не пришли. |
| `LastGeminiRawResponsePreview` | `ПоследнийСыройОтветGeminiПревью` | Строка | Превью сырого JSON-ответа Gemini (до 4096 байт, затем `...(truncated)`). Полезно для диагностики ошибок парсинга/формата ответа. |

### Коды `ПоследняяОшибка` (`LastErrorCode`)

| Код | Значение |
|-----|----------|
| `0` | Успех. |
| `1` | Пустые данные PDF или изображения. |
| `4` | Некорректная строка Base64. |
| `5` | Неверный тип аргумента: ожидались двоичные данные (BLOB). |
| `6` | Неверный тип аргумента: ожидалась строка UTF-8 (Base64 для PDF/изображения или текст для `GenerateGeminiText`). |
| `7` | Не задан ключ API (`КлючAPIAIStudio` / `AIStudioApiKey`). |
| `8` | Ошибка Gemini (сеть, HTTP, ответ API). |
| `9` | Пустой текст запроса (для `GenerateGeminiText` / `СгенерироватьТекстИИ`). |
| `10` | Текст запроса превышает лимит (~2 МиБ UTF-8). |
| `99` | Прочая ошибка; детали — в `ТекстПоследнейОшибки` / поле `error` JSON. |

## Методы

Платформа отдаёт имя в зависимости от языка компоненты: **англ.** (`lang=0`) / **рус.** (`lang=1`).

| Англ. (`lang=0`) | Рус. (`lang=1`) | Описание |
|------------------|-----------------|----------|
| `ParsePrimaryDocumentPdf` | `РазобратьПервичныйДокументPdf` | Gemini; PDF как **BLOB** (совместимость со старым кодом). |
| `ParsePrimaryDocumentPdfBase64` | `РазобратьПервичныйДокументPdfBase64` | То же; PDF в **Base64** (UTF-8). |
| `ParsePrimaryDocumentPdfGemini` | `РазобратьПервичныйДокументPdfИИ` | То же; PDF как BLOB. |
| `ParsePrimaryDocumentPdfGeminiBase64` | `РазобратьПервичныйДокументPdfИИBase64` | То же; Base64. |
| `ParsePrimaryDocumentImageGemini` | `РазобратьПервичныйДокументИзображениеИИ` | Одно изображение как BLOB. |
| `ParsePrimaryDocumentImageGeminiBase64` | `РазобратьПервичныйДокументИзображениеИИBase64` | То же; Base64. |
| `GenerateGeminiText` | `СгенерироватьТекстИИ` | Один аргумент — **строка UTF-8** (промпт); ответ без схемы первички — **обычный текст** модели. |
| `GetSupportedGeminiModels` | `ПолучитьПоддерживаемыеМоделиGemini` | Без параметров: JSON каталога моделей (**2.5+**, превью **3.x**). |
| `UseFastGeminiTimeoutsProfile` | `ИспользоватьБыстрыйПрофильТаймаутовGemini` | Без параметров: установить быстрый профиль таймаутов (`GeminiReceiveTimeoutMs=20000`, `GeminiTotalDeadlineMs=30000`). Возвращает JSON с применёнными значениями. |

**Возвращаемое значение:** для методов разбора первички — строка UTF-8: JSON от модели или `{"error":"..."}`; для `GenerateGeminiText` / `СгенерироватьТекстИИ` при успехе — **произвольный текст** ответа, при ошибке — `{"error":"..."}`.

При кодах **`5`** и **`6`** дополнительно выбрасывается **исключение** (неверный тип параметра).

После вызовов Gemini (`РазобратьПервичныйДокумент*`, `РазобратьПервичныйДокументИзображениеИИ*`, `GenerateGeminiText`) статистика токенов доступна в свойствах `LastPromptTokens` / `LastOutputTokens` / `LastTotalTokens` и в `LastGeminiUsageJson`. Для отладки сырого ответа используйте `LastGeminiRawResponsePreview` / `ПоследнийСыройОтветGeminiПревью`.

## Формат успешного JSON

Задаётся промптом к Gemini; ориентировочно: `counterparty`, `contract`, `invoiceNumber`, `documentDate`, `bankDetails`, `lineItems`, `rawText` и др. Точный набор полей зависит от ответа модели.

## Примеры для 1С (BSL)

```bsl
ИмяКомпоненты = "VisualRecognition";
ПутьКDll = "C:\Путь\К\visualAddInWin64.dll";

Если Не ПодключитьВнешнююКомпоненту(ПутьКDll, ИмяКомпоненты, ТипВнешнейКомпоненты.Native) Тогда
    ВызватьИсключение "Не удалось подключить внешнюю компоненту";
КонецЕсли;

Компонента = Новый("AddIn.VisualRecognition.VisualRecognition");
Компонента.КлючAPIAIStudio = КлючИзБезопасногоХранилища;
Компонента.МодельGemini = "gemini-3-flash-preview";
// Быстрый профиль таймаутов (20с/30с):
Компонента.ИспользоватьБыстрыйПрофильТаймаутовGemini();
// Либо вручную:
// Компонента.ТаймаутПолученияGeminiМс = 25000;
// Компонента.ОбщийДедлайнGeminiМс = 40000;

// Любой из вариантов — вызов Gemini:
JsonСтрока = Компонента.РазобратьПервичныйДокументPdf(ДвоичныеДанныеФайла);
// JsonСтрока = Компонента.РазобратьПервичныйДокументPdfИИ(ДвоичныеДанныеФайла);
// JsonСтрока = Компонента.РазобратьПервичныйДокументPdfBase64(Base64Строка);

Если Компонента.ПоследняяОшибка <> 0 Тогда
    ВызватьИсключение Компонента.ТекстПоследнейОшибки;
КонецЕсли;
```

Скан отдельным файлом (не PDF):

```bsl
JsonСтрока = Компонента.РазобратьПервичныйДокументИзображениеИИ(ДвоичныеДанныеКартинки);
```

## Обработка ошибок

1. Проверяйте **`ПоследняяОшибка` = 0**.
2. При ненулевом коде показывайте **`ТекстПоследнейОшибки`**.
3. В строке результата может быть JSON с полем `error` — согласуйте с кодом в свойствах.

## История версий

Номер версии задаётся в **`CMakeLists.txt`** (`project(... VERSION ...)`) и доступен в 1С в свойстве **`Версия`** / **`Version`**. В репозитории для релизов используются теги вида **`v2.0.5`**.

### 2.0.5

- Актуализирована версия компоненты в сборке (`PROJECT_VERSION`) и документации репозитория.

### 2.0.4

- Добавлена публикация статистики токенов Gemini из `usageMetadata` в свойства компоненты:
  - `LastPromptTokens` / `ПоследниеВходящиеТокены`,
  - `LastOutputTokens` / `ПоследниеИсходящиеТокены`,
  - `LastTotalTokens` / `ПоследниеВсегоТокенов`,
  - `LastGeminiUsageJson` / `ПоследняяСтатистикаТокеновJSON`.
- Статистика обновляется после вызовов Gemini (PDF, изображение, текст) и доступна без изменения формата возвращаемых значений методов.

### 2.0.3

- В JSON ответа разбора первички добавлены поля **`documentTitle`** (название документа на бланке) и **`documentType`**.
- Для **`documentType`** введены значения: `Счет`, `ПриходнаяНакладная`, `РасходнаяНакладная`, `АктВыполненныхРабот`, `Неопределено`.
- Тип документа определяется по названию/заголовку в промпте; дополнительно реализована пост-обработка в C++: если модель вернула пусто или `Неопределено`, выполняется эвристическая попытка классификации по `documentTitle`.

### 2.0.2

- Метод **`GenerateGeminiText`** / **`СгенерироватьТекстИИ`**: произвольный текстовый запрос к Gemini **без** JSON-схемы первички; при успехе возвращается **обычный текст** (UTF-8), при ошибке — JSON с полем `error`, коды **`ПоследняяОшибка`** **`9`** (пустой промпт), **`10`** (промпт длиннее ~2 МиБ UTF-8).
- Для этого метода: **сохранение форматирования** — в запросе передаётся `systemInstruction` (не схлопывать переносы/отступы/списки без явной просьбы пользователя); из ответа API собираются **все** текстовые `parts` первого кандидата, а не только первая часть.

### 2.0.1

- Регистрация методов: первый псевдоним — **английский** (`lang=0`), второй — **русский** (`lang=1`); см. таблицу в разделе «Методы».
- Каталог моделей Gemini: метод **`GetSupportedGeminiModels`** / **`ПолучитьПоддерживаемыеМоделиGemini`** (JSON с `models`, `defaultModelId`, `namingNote` и т.д.).
- Обновлён перечень идентификаторов моделей (линейка **2.5+**, превью **3.x**; без Gemini 1.x).

### 2.0.0

- **Несовместимо с 1.x:** удалены локальное извлечение текста из PDF (MuPDF/PDFium), эвристический разбор первички без ИИ и методы `ИзвлечьТекстИзPdf` / `ИзвлечьТекстИзPdfBase64`.
- Разбор первички только через **Gemini**; пары методов с именами `РазобратьПервичныйДокументPdf` / `РазобратьПервичныйДокументPdfИИ` (и Base64) ведут на один и тот же конвейер.
- Сборка без зависимости от **pdfium.dll** / **MuPDF**.

### До 2.0.0 (ветка 1.x)

- Локальный текстовый слой PDF и разбор первички на стороне компоненты; отдельно — вызовы **Gemini** с суффиксом **ИИ** в имени метода.
